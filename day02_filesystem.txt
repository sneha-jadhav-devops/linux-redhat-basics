DAY 2 - FILE & DIRECTORY MANAGEMENT

Why file and directory commands are important?
In Linux computers, all is done through files and directories.
These commands are used by DevOps engineers every day to handle applications, systems, and services , logs, configs, and deployments.


Commands learned on Day 2:

1. cd
Used to change directory.
Example:
cd /home/user

2. mkdir
Creates a new directory.
Example:
mkdir logs

3. rmdir
Deletes an empty directory.
Example:
rmdir old_logs

4. touch
Creates an empty file.
Example:
touch app.txt

5. cp
Copies files or directories.
Example:
cp app.txt backup.txt

6. mv
Moves or renames files/directories.
Example:
mv app.txt app_backup.txt

7. rm
Deletes files or directories.
Example:
rm file.txt
rm -r folder_name
